# Skill: Apps, Tools, And Libs

## Use When

- adding or editing apps
- adding or editing tools
- organizing shared or app-local libraries

## Steps

1. Keep apps under `apps/`.
2. Keep tools under `tools/`.
3. Keep shared reusable libraries under root `libs/`.
4. Use Alteran command surfaces for app and tool work where appropriate.
5. Preserve the single `@libs/...` import model when describing or editing library usage.

## Done Checklist

- [ ] Apps, tools, and libs stay in the expected Alteran layout.
- [ ] Shared vs app-local libraries are described honestly.
- [ ] No alternate project structure was invented.
